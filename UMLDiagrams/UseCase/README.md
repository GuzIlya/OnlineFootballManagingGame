## 1 Глоссарий<a name="1"></a>

| Термин| Определение|
|:---|:---|
| Гость | Неавторизовавшийся в системе пользователь |
| Зарегистрированный пользователь | Авторизовавшийся в системе пользователь |
| Администратор | Авторизовавшийся в системе пользователь, имеющий права на редактирование информации |
| Информация | Все представенные на сайте данные. Можно разделить на: изменяемые администратором(результаты матчей, покупка игроков), изменяемые пользователем(формирование состава) и изменяемые системой(таблица лидеров)
| Профиль| Информация о балансе и имени авторизованного пользователя, отображаемая в верхней части главной страницы сайта |
| Сезон | Определенный промежуток времени, установленный игрой, в течение которого происходит подсчет очков пользователей(с началом нового сезона таблица лидеров обнуляется) |
| Тур | Событие, по результатам которого пользователи набирают очки |

<p align="justify"> Актёрами являются гость, зарегистрированный пользователь и администратор.</p>

## 3 Варианты использования<a name="3"></a>
### 3.1 Зарегистрироваться <a name="3.1"></a>
**Описание.** Вариант использования "**Зарегистрироваться**" позволяет гостю создать свой профиль.<br>
**Поток выполнения:**
1. Пользователь нажимает клавишу "Sign up" на основой странице сайта.
2. Система отображает страницу регистрации, содержащую поля ввода имена, пароля и подтвержедения пароля.
3. Пользователь заполняет все поля ввода и нажимает на кнопку "Sign up".
4. Система проверяет уникальность имени. Если данное имя уже есть среди зарегистрированных пользователей, выполняется п.5. Если имя уникальное, выполняется п.6 потока.
5. Слева от кнопки "Sign up" появляется сообщение о том, что данное имя уже используется. Процесс регистрации возвращается в п.3 потока.
6. Система проверяет совпадение паролей. Если пароли не совпадают, выполняется п.7. Если совпадают, выполняется п.8 потока.
7. Слева от кнопки "Sign up" появляется сообщение о том, что пароли не совпадают. Процесс регистрации возвращается в п.3 потока.
8. Система добавляет пользователя в базу данных.
9. Система отображает сообщение об успешной регистрации.
10. Конец.
---
### 3.2 Авторизоваться<a name="3.2"></a>
**Описание.** Вариант использования "**Авторизоваться**" позволяет гостю войти в свой профиль.<br>
**Поток выполнения:**
1. Пользователь нажимает клавишу "Log in" на основной странице сайта.
2. Система отображает страницу авторизации, содержащую поля ввода имена и пароля.
3. Пользователь заполняет все поля ввода и нажимает на кнопку "Sign up".
4. Система проверяет введенные данные с данными из базы данных . Если совпадение не обнаружено, выполняется п.5. Если обнаружено, выполняется п.6 потока.
5. Слева от кнопки "Log in" появляется сообщение о том, что данные введены неверно. Процесс авторизации возвращается в п.3 потока.
6. Система отображает основную страницу сайта с авторизованным пользователем.
7. Конец.
---
### 3.3 Просмотреть таблицу лидеров<a name="3.3"></a>
**Описание.** Вариант использования "**Просмотреть таблицу лидеров**" позволяет гостю или зарегистрированному пользователю увидеть информацию о текущей таблице лидеров.<br>
**Поток выполнения:**
1. Пользователь нажимает на вкладку "Rating" на основной странице сайта.
2. Система отображает страницу таблицы лидеров, содержащую поля имен пользователей, их место с таблице и количество заработанных за сезон очков.
3. Конец.
---
### 3.4 Сформировать состав<a name="3.4"></a>
**Описание.** Вариант использования "**Сформировать состав**" позволяет зарегистрированному пользователю сформировать стартовый состав среди доступных игроков.<br>
**Поток выполнения:**
1. Пользователь нажимает на вкладку "My Squad" на основной странице сайта.
2. Система отображает страницу состава пользователя, содержащую combobox выбора формации, 11 кнопок добавления игрока в состав, список доступных игроков.
3. Пользователь выбирает формацию состава, выбрав необходимую в combobox выбора состава.
4. Система отображает соответствующую формацию как определенное расположение 11 кнопок добавления игрока.
5. Пользователь нажимает на одну из кнопок добавления игрока.
6. Система подсвечивает из списка доступных игроков тех игроков, чья позиция позволяет расположить данного игрока в выбранной кнопке.
7. П.5-6 повторяются для всех 11 кнопок. 
8. Пользователь нажимает на кнопку "Save".
9. Система сохраняет сформированный состав в базе данных и регистрирует его на предстоящий тур, в случае, если даный игрок уже регистрировал состав, предыдущий состав перезаписывается новым.
10. Конец.
---
### 3.5 Купить игроков<a name="3.5"></a>
**Описание.** Вариант использования "**Купить игроков**" позволяет зарегистрированному пользователю купить игроков.<br>
**Поток выполнения:**
1. Пользователь нажимает на вкладку "Transfer market" на основной странице сайта.
2. Система отображает страницу покупки игроков, содержащую combobox выбора позиции игрока, combobox выбора нации игрока, список найденных игроков.
3. Пользователь выбирает позицию игрока, выбрав необходимую в combobox выбора позиции игрока.
4. Пользователь выбирает нацию игрока, выбрав необходимую в combobox выбора нации игрока.
5. Пользователь нажимает на кнопку "Search".
6. Система ищет игроков в базе данных, соответсвующих данным критерия поиска и отображет их списке найденных игроков.
7. Пользователь наводит мышь на необходимого игрока и списка доступных.
8. Система отображает стоимость игрока и кнопку покупки "Buy" справа от имени данного игрока.
9. Пользователь нажимает на кнопку "Buy".
10. Система считавает стоимость игрока из баланса зарегистрированного польщователя и добавляет данного игрока в список доступых для пользователя во вкладке "My Squad".
11. Пункты 3-10 повторяются до того момента, как пользователь не завершит процесс покупки игроков. 
12. Конец.
---
### 3.6 Просмотреть результаты матчей<a name="3.6"></a>
**Описание.** Вариант использования "**Просмотреть результаты матчей**" позволяет зарегистрированному пользователю просмотреть результаты прошедших матчей.<br>
**Поток выполнения:**
1. Пользователь нажимает на вкладку "Recent matches" на основной странице сайта.
2. Система отображает страницу результатов матчей, содержащую список прошедших матчей, а также их счет.
3. Конец.
---
### 3.7 Изменить информацию<a name="3.7"></a>
**Описание.** Вариант использования "**Изменить информацию**" позволяет администратору изменить информацию о прошедших матчах и стоимости игроков.<br>
**Поток выполнения:**
1. Администратор нажимает на вкладку "Recent matches" на основной странице сайта.
2. Система отображает страницу результатов матчей, содержащую список прошедших матчей, их счет, и кнопку "Add".
3. Администратор нажимает на кнопку "Add".
4. Система отображает форму для заполнения, содержащую поля выбора команд, поле счета и кнопку "Confirm".
5. Пользователь заполняет соответствущие поля и нажимает на кнопку "Confirm".
6. Система добавляет данный матч в список прошедших и производит подсчет очков для всех пользователей, чьи игроки учавствовали в данных матчах.
7. Конец.
---
### 3.8 Выйти из системы<a name="3.8"></a>
**Описание.** Вариант использования "**Выйти из системы**" позволяет администратору или зарегистрированному польщователя выйти из системы.<br>
**Поток выполнения:**
1. Пользователь нажимает на кнопку "Sign out" на основной странице сайта.
2. Система делает пользователя гостем.
3. Система отображает основную страницу сайта для гостя.
4. Конец
